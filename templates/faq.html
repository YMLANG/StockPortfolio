The first issue is that the stock/symbol exists, but the API returns an error (e.g. API crashed). To solve this, simply refresh the page.

The second issue would be the stock/symbol exists in the database, but not in the API, hence API retuns an error. There are no solution for this error, instead search for another stock/symbol.

The final issue would be that the API is offline/down, hence API returns an error. To solve this, simply wait for additional 3 ~ 15 minutes for the API to restart.

Ans: We use supervised learning to predict tomorrow's trend for each individual stock. The model we choose is Support Vector Machine classifier. We train the SVM for each individual stock based on all the historical data and classify it as rise or fall. In other words, given the data at time t, we train the model to predict trend(rise/fall) at time t+1.
If the model predict "N/A", it means our model fail to learn this stock. The accuracy is around 50~65% and it's various from stock to stock.

The features are 23 technical indicators which the traders used in quantatitive analysis:
'MA_5', 'MA_7', 'MA_10', 'EMA_5', 'EMA_12', 'EMA_26', 'Momentum_1', 'Momentum_4', 'MACD', 'MACDsignal', 'MACDdiff', 'TSI_7_4', 'BollingerB_5', 'Bollinger%b_5', 'OBV_1', 'Mass Index', 'Force_1', 'SO%k', 'SO%d_3', 'Chaikin', 'Disparity_5', 'Disparity_10', 'STD_7'

The method is developed based on these two papers:
http://lcsd2.svms.org/finance/Kim2003.pdf
http://sci2s.ugr.es/keel/pdf/specific/articulo/ESA00.pdf

The formulas of technical indicators follows (with tiny modifications):
https://www.quantopian.com/posts/technical-analysis-indicators-without-talib-code

Pipline
Raw Data: daily historical prices of each individual stock (OHLCV) (source: Quandl)
Preprocess: calculate 27 technical indicators as features and then scale the dataset to zero variance
Feature Selection: use cross validation to eliminate 4 features which doesn't significant impact on classification
Model: SVM with Linear Kernel (speed & accuracy trade-off)
